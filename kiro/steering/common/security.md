# セキュリティガイドライン

## コミット前チェックリスト

- [ ] ハードコードされたシークレットがない（APIキー、パスワード、トークン）
- [ ] すべてのユーザー入力が検証済み
- [ ] SQLインジェクション防止（パラメータ化クエリ）
- [ ] XSS防止（HTMLサニタイズ）
- [ ] CSRF保護が有効
- [ ] 認証/認可が検証済み
- [ ] エラーメッセージが機密データを漏洩しない

## シークレット管理

- ソースコードにシークレットをハードコードしない
- 環境変数またはシークレットマネージャーを使用
- 起動時に必要なシークレットの存在を検証
- 漏洩の可能性があるシークレットはローテーション
